5. Generative Models Today

The previous experiment used Gaussian distributions because they are easy to visualize. But the same idea appears in modern generative models.

Consider a Conditional Variational Autoencoder, or CVAE, trained on images. Instead of directly learning a classification boundary, it learns how images are generated for each class.

$$ P(x|y) $$

More precisely, the model introduces a latent variable \(z\), which captures hidden factors of variation such as shape, pose, thickness, texture, or style.

$$ P(x|y) = \int P(x|z,y)P(z)\,dz $$

To classify a new image \(x\), we can ask which class would most likely have generated it:

$$ \hat{y} = \arg\max_c P(x|y=c)P(y=c) $$

This is the same principle as the Gaussian experiment. The ellipses are gone, but the logic is unchanged: learn a model of each class, then classify by comparing how well each class explains the input.

A CVAE does not need to learn a decision boundary directly. The boundary emerges from comparing class-conditional likelihoods.

A similar idea appears in foundation-model pretraining. A model may first be trained with a generative objective, such as predicting missing parts of the data.

$$ \text{pretrain by learning structure in } x $$

After this pretraining stage, the learned representation can be adapted to a specific downstream task, such as image classification, diagnosis, or sequence labeling.

The important point is not that one model magically solves every task. The point is more precise: a generative pretext task can force the model to learn useful structure before it is trained for one target task.

Generative learning can be useful even when the final goal is discriminative.

This gives a modern version of the article's main thesis: learning how data is produced can provide a richer starting point than learning only how to separate labels.

But richer does not always mean better. In practice, choosing between generative and discriminative models depends on the task, the data, and the assumptions we are willing to make.
